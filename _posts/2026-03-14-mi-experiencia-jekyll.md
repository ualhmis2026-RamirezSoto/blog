\# Mi experiencia con Jekyll y el desarrollo colaborativo



\*\*Fecha\*\*: 14 de marzo de 2026

\*\*Autor\*\*: Soto



\## Primeros pasos con Jekyll



Esta es mi primera vez trabajando con Jekyll y debo decir que me ha sorprendido lo fácil que es crear un blog estático. Al principio me costó entender la estructura de directorios, pero una vez que lo comprendes, todo fluye.



\## Lo que más me ha gustado



\- \*\*La simplicidad\*\*: Escribir en Markdown y que se convierta automáticamente a HTML es genial

\- \*\*La organización\*\*: Los posts se ordenan por fecha automáticamente

\- \*\*La personalización\*\*: Podemos modificar el diseño fácilmente



\## Trabajo en equipo con Git



Estamos utilizando GitHub Flow con ramas por característica. Yo he creado mi rama `soto-feature` para trabajar en mi perfil y este post. Luego haré un pull request para que Bruno (el responsable) revise mis cambios y los fusione con la rama principal.



\## Comandos útiles que he aprendido



```bash

\# Ver las ramas disponibles

git branch



\# Crear y cambiar a nueva rama

git checkout -b mi-rama



\# Ver el estado de mis cambios

git status



\# Añadir cambios

git add .



\# Hacer commit

git commit -m "Descripción de mis cambios"



\# Subir cambios a GitHub

git push origin mi-rama
